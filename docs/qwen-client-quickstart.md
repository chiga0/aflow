# AgentFlow 客户端快速上手

本文帮助真实用户从零开始完成第一个任务，并在日常使用中高效协作。你不需要理解 Worker、Executor、Adapter 或 Workflow Engine 等实现细节。

## 三步开始

1. **打开 Client**：访问 `https://<你的域名>/cloud-agents/#/`，使用管理员提供的邮箱和密码登录。
2. **描述目标**：在左侧点击"新建会话"，在输入框中用自然语言写清你要完成的事情，例如"帮我审查 src/auth.py 的权限逻辑并给出修改建议"。
3. **发送**：点击发送即可。系统自动选择执行策略，**默认无需手动选择 Mode 或 Adapter**。

任务提交后，Chat 区域会依次展示 Agent 的阶段简报和最终结果。你可以在左侧会话列表随时查看进度。

## 持续会话

会话是 AgentFlow 的核心工作单元，支持多轮交互：

- **追加上下文**：任务运行中或完成后，可在输入框继续补充要求，例如"额外检查 SQL 注入风险"。
- **调整方向**：如果 Agent 的输出不完全符合预期，直接说明你希望调整的部分。
- **停止任务**：运行时输入框会变为"停止"按钮，点击可终止当前执行。

当目标发生根本变化、需要更换工作目录或执行策略时，建议新建会话而非在旧会话中大幅转向。

## 多 Agent 与 Canvas

复杂任务会被自动拆分为多个有依赖关系的 Agent 协作完成；当前本地执行器按依赖顺序执行。你不需要手动编排。

- **Chat 视图**：只显示“已生成执行计划，将由 N 个 Agent 协作完成”等摘要信息，不暴露 Agent 内部通信。
- **Canvas 视图**：桌面端右侧 Canvas 提供计划、Agent、变更、文件、产物、流程、验收和活动 8 个结构化视图；移动端通过横向标签按需切换。
- **按需查看**：可从 Chat 中的计划、产物或执行记录入口定位到对应 Canvas 内容，不影响其他会话继续运行。
- **双向定位**：Chat 中的阶段简报可深链到对应计划或 Agent；Agent 节点也可定位回相关简报。

## 移动审批

当 Agent 需要执行高风险操作（如修改生产配置、删除文件）时，系统会生成审批请求：

- **桌面端**：Chat 中出现审批卡片，展示意图、证据摘要和影响范围，点击"批准"或"拒绝"。
- **移动端**：移动决策台只同步状态快照与审批请求，可查看关键证据并批准、拒绝、暂停或要求修改。
- **跨端保护**：同一审批在桌面和移动端同时出现时，服务端保证只有一个决策生效，不会重复执行。

审批卡片始终包含四个要素：**意图**（Agent 想做什么）、**证据**（关键 Diff 或命令）、**影响**（风险级别和可逆性）、**决策**（批准 / 拒绝）。

## 安全边界

- **访问隔离**：会话、消息和审批按已认证用户校验访问权；移动端不会下载完整工作区。
- **推送安全**：移动端通知只携带状态摘要，不包含代码、密钥或完整命令。
- **跨端同步**：在线状态优先通过 SSE 更新，断线后使用轮询恢复；离线期间禁止提交审批，避免重复决策。
- **操作审计**：会话消息、任务状态、Agent 阶段结果和审批决策写入规范事件。
- **敏感信息保护**：仍应避免把真实凭据写入任务描述和仓库文件。

> 当前版本尚未完成强多租户隔离，也不自动保证终端输出和产物中的脱敏处理。请在提交任务前自行审查内容。

## 失败恢复

任务失败时，系统会展示人类可读的原因和可执行选项：

| 情况 | 你可以做什么 |
| --- | --- |
| 单个 Agent 失败，整体可继续 | 选择仅重试失败步骤，或接受部分结果 |
| 任务完全失败 | 查看失败摘要后点击"重试"，或修改目标重新提交 |
| 网络断开 | 系统提示连接状态；远端任务可能仍在运行，恢复连接后自动同步，不会重复发送消息或重复审批 |
| 执行单元离线 | 管理员可在 Admin 中检查 Worker 状态；用户可等待恢复后重试 |

错误信息始终回答三个问题：**发生了什么、当前是否仍在执行、你可以做什么**。不会出现裸错误码或原始异常堆栈。
